# <a name="developer39s-guide-analysis-services"></a>개발자&#39;가이드 (Analysis Services)
    
## <a name="in-this-section"></a>섹션 내용  
 [테이블 형식 모델 프로그래밍](tabular-model-programming-compatibility-levels-1050-1103/tabular-model-programming-for-compatibility-levels-1050-through-1103.md)  
 테이블 형식 모델을 정의하는 CSDL 확장 프로그램을 XML 구문으로 설명합니다. 여기에는 테이블 형식 개체 모델에 대한 정보도 포함됩니다.  
  
 [데이터 마이닝 프로그래밍](dev-guide/data-mining-programming.md)  
 데이터 마이닝 개체를 포함하는 솔루션을 구축하는 방법에 대해 설명합니다.  
  
 [다차원 모델 프로그래밍](multidimensional-models/multidimensional-model-programming.md)  
 다차원 모델 개체를 사용자 지정 솔루션에 통합하는 방법 및 개발 태스크에 대해 설명합니다.  
  
 [Analysis Services 스키마 행 집합](https://docs.microsoft.com/bi-reference/schema-rowsets/analysis-services-schema-rowsets)  
 서버 상태, 서버 작업 및 데이터베이스 개체에 대한 정보를 제공하는 스키마 행 집합에 대해 설명합니다.  
  
 [Analysis Services Scripting Language &#40;ASSL&#41; 참조](https://docs.microsoft.com/bi-reference/assl/analysis-services-scripting-language-assl-for-xmla)  
 XMLA에 대한 ASSL 확장 프로그램에 대해 설명합니다. ASSL은 XMLA 사양을 보완하는 Analysis Services 다차원 모델에 대한 데이터 정의 및 조작 언어를 제공합니다.  
  
 [XMLA&#40;XML for Analysis&#41; 참조](https://docs.microsoft.com/bi-reference/xmla/xml-for-analysis-xmla-reference)  
 XMLA가 사용자 지정 솔루션에 적용되는 방법을 이해하는 데 도움이 될 수 있는 XMLA 개념에 대해 설명합니다. 또한 XMLA 1.1 사양과의 호환성 수준에 대해서도 설명합니다.
