# <a name="ado-glossary-terms"></a>ADO 용어
이 항목에서는 ADO와 관련 된 용어를 정의합니다.

## <a name="a"></a>A
 절대 URL는 정규화 된 URL 인터넷 또는 인트라넷에 있는 리소스의 위치를 지정 하는입니다. 참고 항목 *URL* 하 고 *상대 URL*합니다.

 ActiveX 컨트롤이 디자인 타임 또는 런타임 시각적 요소를 종종 있는 자동 등록, in process COM 구성 요소입니다. ActiveX 컨트롤에는 또한 Microsoft Internet Explorer와 같은 액티브 문서 컨테이너를 통신할 수가 있습니다.

 ADISAPI (고급 데이터 인터넷 서버 응용 프로그래밍 인터페이스)는 ISAPI DLL에 Automation 제어, 레코드 집합 마샬링 및 MIME 패키징의 구문 분석을 제공 합니다. 인터넷 정보 서비스 (IIS)에서 제공 하는 API를 통해 ADISAPI 구성 요소가 작동 합니다. 참고 항목 *ISAPI*합니다.

 집계 함수를 쿼리, COUNT, AVG 또는 모든 행을 사용 하 여 테이블의 열에 값을 계산 하는 STDEV와 같은 함수입니다. 식 작성 및 프로그래밍에서는 다양 한 통계를 확인 하려면 SQL 집계 함수 (위에 나열 된 3 포함) 및 도메인 집계 함수를 사용할 수 있습니다.

 별칭 대체 이름 열 또는 SQL SELECT 문에서 자주 보다 짧거나 의미 있는 식에 부여 합니다. 예를 들어 BobSales은 다음 SELECT 문에서 별칭: "SalesDB에서 BobSales로 쓰기 매출을 선택 합니다."입니다. DataControl 개체에 바인딩 컨트롤에 열을 동적으로 할당 하는 별칭을 사용할 수 있습니다.

 아파트 스레드는 COM 스레딩 모델이 개체에 대 한 모든 호출 스레드에서 발생 합니다. COM은 아파트 스레딩 동기화 하 고 호출을 마샬링합니다. 참고 항목 *COMmddefcom*합니다.

 작업이 완료 될 때까지 기다리지 않고 호출 프로그램에 제어를 반환 하는 작업을 비동기 작업입니다. 작업이 완료 되 면 전에 코드 실행이 계속 됩니다. 참고 항목 *동기 작업*합니다.

 CLSID (클래스 식별자)는 범용 고유 식별자 (UUID) COM 구성 요소를 식별 하는입니다. 각 COM 구성 요소는 다른 응용 프로그램에서 로드할 수 있도록 Windows 레지스트리에서 CLSID에 있습니다. 참고 항목 *ProgID*하십시오 *COM*합니다.

 로 표현 될 수 있습니다 하는 조건 매개 변수 ">" (보다 큼), "\<" (보다 작음), "=" (같음), "> =" (보다 크거나 같음), "< =" (작거나 같음), "<>" (다음과 같지 않은 경우) 또는 "좋아요" (패턴 일치).

 구성 요소 데이터와 코드를 캡슐화 하 고 잘 지정 된 집합이 공개적으로 사용 가능한 서비스를 제공 하는 개체입니다.

 많은 개별 파일을 하나의 실제 파일에 함께 바인딩됩니다. 복합 파일의 각 개별 파일을 하나의 실제 파일 처럼 액세스할 수 있습니다.

 상수는 변경 되지 않는 숫자 또는 문자열 값입니다. 실제 값 대신 코드에서 예를 들어, 명명 된 ADO 열거 (열거 상수)를 사용할 수 있습니다 **adUseClient** 값이 3 인 상수입니다. (Const adUseClient = 3). 참고 항목 *열거형*합니다.

 database 요소 레코드 탐색, 데이터의 업데이트 및 다른 사용자가 데이터베이스에 대 한 변경 내용의 표시 유형을 제어 하는 커서입니다.

 트리는 구조 (노드) 요소 간의 계층 관계를 나타내는입니다. (루트) 트리의 최상위 수준에서 노드 하나 있습니다. 루트 아래에 여러 자식이 있을 수 있습니다. 각 자식에 다른 자식에 따라서를 트리와 같이 분기의 부모 수도 있습니다. 문서와 다른 폴더를 포함 하는 폴더에는 트리 구조의 일반적인 예입니다. 계층, 노드, 루트, 자식, 상위를 참고 하십시오.
